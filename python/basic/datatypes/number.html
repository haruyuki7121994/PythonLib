Number Data Type

Python hỗ trợ kiểu integer float complex Chúng được định nghĩa là các Class int, float, complex trong Python.

integer và float được phân tách nhau bằng có hoặc không có dấu thập phân. Ví dụ: 5 là integer, 5.0 là float.

complex được có syntax là x + yj trong đó x là phần thực và yj là phần ảo.

Chúng ta có thể sử dụng hàm type() để biết biến hoặc giá trị thuộc về lớp nào. Hàm isinstance() để kiểm tra xem nó có thuộc một lớp cụ thể nào không.

a = 5

print(type(a))
>>> <class 'int'>

print(type(5.0))
>>> <class 'float'>

c = 5 + 3j
print(c + 3)
>>> (8 + 3j)

print(isinstance(c, complex))
>>> True

Trong khi integer có thể có độ dài bất kỳ. float chỉ chính xác đến 15 chữ số thập phân (vị trí thứ 16 là không chính xác).

Python cũng hỗ trợ biểu diễn cơ số 2 binary, cơ số 16 hexadecimal, cơ số 8 octal theo bảng tiền tố dưới đây:

Number System Prefix
Binary '0b' or '0B'
Octal '0o' or '0O'
Hexadecimal '0x' or '0X' 
# Output: 107
print(0b1101011)

# Output: 253 (251 + 2)
print(0xFB + 0b10)

# Output: 13
print(0o15)

Type Conversion

Chúng ta có thể chuyển đổi 1 kiểu số thành một kiểu số khác. Đây được gọi là coercion (sự ép buộc).

Các phép toán như + - sẽ chuyển đổi số có kiểu integer sang kiểu float hoàn toàn tự động. Nếu một trong các toán hạng là float

>>> 1 + 2.0
3.0

Chúng ta có thể thấy rằng số 1 bị ép thành số 1.0 để cộng và kết quả là kiểu float

Chúng ta cũng có thể sử dụng các hàm int() float() complex() để chuyển đổi giữa các kiểu số. Các hàm này có thể chuyển đổi từ chuỗi.

>>> int(2.3)
2
>>> int(-2.8)
-2
>>> float(5)
5.0
>>> complex('3+5j')
(3+5j)

Module Decimal

Class float được tích hợp sẵn trong Python thực hiện một số phép tính không giống như chúng ta suy nghĩ. Ví dụ tổng 1.1 + 2.2 = 3.3

>>> (1.1 + 2.2) == 3.3
False

Kiểu float được thực hiện trong phần cứng máy tính dưới dạng số nhị phân ví máy tính chỉ hiểu được số nhị phân. Vì lý do này, hầu hết các số thập phân mà chúng ta biết không thể lưu trữ một cách chính xác trong máy tính. Ví dụ: chúng ta không thể biểu diễn phân số 1/3 dưới dạng phân số thập phân. Điều này sẽ cho 0.3333333... dài vô hạn và máy tính chỉ có thể lưu hữu hạn.

Nó chỉ ra rằng phân số thập phân 0.1 sẽ được lưu dưới dạng nhị phân vô hạn 0.000110011001100110011... và máy tính chỉ có thể lưu hữu hạn. Dẫn đến số được lưu gần đúng 0.1 nhưng không bao giờ bằng nhau. Đó là hạn chế của phần cứng máy tính chứ không phải lỗi của Python

Để khắc phục vấn đề này, chúng ta sử dụng module Decimal Trong khi các số float được lưu hữu hạn 15 chữ số thì module này có độ chính xác do người dùng thiết lập. Module này được sử dụng khi chúng ta muốn thực hiện các phép tính thập phân như chúng ta suy nghĩ.

from decimal import Decimal as D

print(D('1.1') + D('2.2'))
>>> 3.3

print(D('1.2') * D('2.50'))
>>> 3.000

Lưu ý: Các phép toàn thực hiện bằng kiểu số float được thực hiện nhanh hơn phép toán Decimal Vì lý do này chúng ta không triển khai Decimal mọi lúc thay vì kiểu float. Vậy khi nào chúng ta sử dụng Decimal thay vì float?

  • Khi tạo ra các ứng dụng tài chính cần số thập phân chính xác
  • Khi muốn kiểm soát độ chính xác
  • Khi muốn triển khai các khái niệm về số thập phân

Module Fractions

Python cung cấp các phép toán liên quan đến phân số thông qua module fractions

Một phân số có tử số và mẫu số, cả 2 đều là số nguyên. Module này có hỗ trợ cho số học hữu tỉ.

import fractions

print(fractions.Fraction(1.5))
>>> 3/2

print(fractions.Fraction(5))
>>> 5

print(fractions.Fraction(1,3))
>>> 1/3

Khi tạo phân số từ số float, chúng ta có thể sẽ nhận được phân số không muốn. Bởi vì số float ko được lưu chính xác. Lúc đó, chúng ta phải khởi tạo phân số từ chuỗi được cung cấp từ module fractions

import fractions

# As float
# Output: 2476979795053773/2251799813685248
print(fractions.Fraction(1.1))

# As string
# Output: 11/10
print(fractions.Fraction('1.1'))

Module Math & Random

Khi muốn thực hiện các phép toán về lượng giác, logarit, xác xuất, thống kê... Python cũng cung cấp 2 module quan trọng để thực hiện các phép toán này là math và random

import math

print(math.pi)
>>> 3.141592653589793

print(math.cos(math.pi))
>>> -1.0

print(math.exp(10))
>>> 22026.465794806718

print(math.log10(1000))
>>> 3.0

print(math.sinh(1))
>>> 1.1752011936438014

print(math.factorial(6))
>>> 720
import random

print(random.randrange(10, 20))

x = ['a', 'b', 'c', 'd', 'e']

# Get random choice
print(random.choice(x))
>>> 18

# Shuffle x
random.shuffle(x)
>>> e

# Print the shuffled x
print(x)
>>> ['c', 'e', 'd', 'b', 'a']

# Print random element
print(random.random())
>>> 0.5682821194654443
